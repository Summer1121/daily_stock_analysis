# 提案：为应用增加一个统一的控制台

## 目标

本项目旨在为该应用增加一个功能强大且用户友好的控制台，用户可以通过这个控制台监控和控制系统的所有核心功能。这将极大地提升系统的可用性和交互性，将一个自动化分析工具转变为一个全面的、可视化的投研与交易平台。

## 背景

当前系统虽然功能强大，但主要通过配置文件和日志进行交互，缺乏一个直观的图形用户界面。用户难以实时监控系统状态、调整策略或进行手动操作。

新的控制台将解决这些痛点，提供一个集中的界面来管理以下功能：

- **实时监控**：实时新闻流、走势分析记录、交易看板（持仓、盈利、汇总）。
- **配置管理**：信息渠道配置、推送配置、策略选择（启用/禁用、切换）。
- **手动操作**：手动执行交易、触发分析任务。
- **策略回测**：允许用户在指定时间区间内，对指定的交易策略进行历史回测。

## 范围

此提案涵盖了控制台的前端和后端开发。

- **前端**：一个单页面应用（SPA），提供丰富的交互体验。
- **后端**：一组新的 API 端点，用于支持前端控制台的各种功能请求。
- **集成**：将新的控制台与现有的分析、交易和通知模块无缝集成。
